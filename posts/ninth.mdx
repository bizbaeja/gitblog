---

### React 만 이용해 만든 일반적입 앱의 페이지의 소스코드를 확인하면,

기본 뼈대 HTML 이 있고, `<div id=”root”></div>` 여기서 React 앱을 읽어들여 렌더링한다. 이 렌더링은 React 에서 이루어진다.

\*\* 리액트는 클라이언트 사이드 자바스크립트 라이브러라라서 모든 렌더링은 클라이언트 쪽에서 일어난다. 사용자 브라우저에서 보이는 것은 서버에서 나오는 것이 아니다.

### 기존 리액트의 불편했던 점

> ‘ 페이지 깜빡임 → 사용자 경험 저해 ’, ‘ 검색엔진 최적화에 방해가 됨 ’

<aside>
💡 예를 들어 페이지가 서버에서 데이터를 받아와서  고양이 간식 판매 목록을  화면에 렌더링 해줘야 한다면, 사용자는  처음에 로딩 상태를 보게 됩니다. (아주 잠깐 페이지가 깜빡이게 됩니다)
이 사이에 데이터를 받아오는 요청이 진행됩니다.

데이터 가져오기는 클라이언트 쪽에서 자바스크립트 코드가 실행될 때만 시작하기 때문입니다.

그리고 발송된 응답의 요청을 기다려야 합니다.

요청한 페이지에 해당 데이터가 아직 포함되지 않았기 때문입니다.

</aside>

---

로그인을 해야만 볼 수 있는 관리 대시보드인 경우에는 문제가 되지 않지만

검색 엔진은 이 대시보드를 절대 보지 못하므로 문제상황이 발생하게 됩니다.

리액트에 도달한 *검색 엔진 크롤러*는 서버에서 들어온 비어있는 초기 HTML 페이지만 보고있습니다.

---

**이럴 때 서버 사이드 렌더링이 필요합니다.**

### NEXTJS 의 이점

서버 사이드 렌더링이 내장되어 있기 때문에, 추가 설정을 하지 않고도, 페이지에 방문했을 때, 서버에서 기본적으로 바로 사전 렌더링을 한다.

→ 검색 엔진 최적화

(검색 엔진이 사용자가 보는 대로 보기 된다)

사용자에게 더 나은 초기 로딩 환경을 제공한다 (초기에 화면이 깜빡이지 않게 된다)

---

NEXTJS 의 소스코드 HTML은 비어있는 뼈대가 아니라 모든 콘텐츠가 해당 HTML 페이지에 있다

→ 서버로부터 받은 것,

\***\* 더 중요한 이점 \*\***

NEXTJS 에서는 클라이언트와 서버 사이드 코드를 혼합한다

- 초기 요청으로 읽어들인 후에도 기본 React 앱과 SPA 를 실행할 수 있다.
- 사용자가 후속 탐색을 할 수 있다.
- 사용자가 페이지를 열어서 탐색할 때, 이런 활동들은 모두 브라우저에서 React가 관리하여 사용자에게 빠른 대화형 인터페이스를 제공할 수 있다.

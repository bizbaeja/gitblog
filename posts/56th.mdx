# 네임스페이스란?

형식의 ‘성’ 으로 생각하면 된다. 클래스에는 형식을 구현하는 코드가 포함되어 있다. 이름 충돌을 방지하기 위해 클래스는 네임스페이슬 구성된다. 수천 개의 클래스가 있으면, ㅋㄹ래스 이름을 재사용해야 하는 경우도 있다.

# .NET 클래스 라이브러리에서 여러 종류의 메서드 호출법

1. 클래스 이름 `Console` 을 입력한다.
2. 멤버 액세스 연산자인 `.` 기호를 추가한다.
3. 메서드 이름인 `WriteLine` 을 추가한다.
4. 괄호 세트 `()` 인 메서드 호출 연산자를 추가한다.
5. 마지막으로 , `Console.WriteLine()` 메서드에서 입력 매개 변수로 출력하려는 값을 여는 괄호와 닫는 괄호 사이에 추가합니다. (예: `“Hello World!”`)

### 다음 예제는 난수를 생성하고 콘솔에 출력하여 주사위 굴리기를 시뮬레이트 합니다.

```csharp
Random dice = new Random();
int roll = dice.Next(1,7);
Console.WriteLine(roll);
```

1. 첫 번째 코드 줄에서는 .NET 클래스 라이브러리에 `System.Random` 클래스의 새 인스턴스를 만들고 새 개체에 대한 참조를 `dice` 변수에 저장합니다.
2. 두 번째 코드 줄에서는 난수의 최소값과 최댓값인 두 개의 매개 변수를 전달하여 `dice` 개체의 `Next()` 메서드를 호출합니다. `Next()` 메서드에 반환된 값을 `roll` 변수에 저장합니다.
3. 세 번째 코드 줄에서는 `WriteLine()` 메서드를 호출하여 `roll` 값을 콘솔에 출력합니다.

> 컴퓨팅에서 상태는 특정 시점의 실행 환경 조건을 설명합니다. 코드가 줄 단위로 실행됨에 따라 값이 변수에 저장됩니다. 실행 중의 모든 시점에서 애플리케이션의 현재 상태는 메모리에 저장된 모든 값의 컬렉션입니다.

> 일부 메서드는 애플리케이션의 현재 상태와 관계 없이 제대로 작동합니다. 즉 , 상태 비저장 메서드는 메모리에 저장된 값을 참조하거나 변경하지 않고 작동할 수 있도록 구현됩니다. 상태 비저장 메서드를 **_정적 메서드_**라고 합니다.

> 예를 들어 Console.WriteLine() 메서드는 메모리에 저장된 값을 사용하지 않습니다. 애플리케이션 상태에 어떠한 영향도 주지 않고 해당 기능을 수행한 후 종료됩니다.

> 그러나 제대로 작동하려면 애플리케이션 상태에 대한 액세스 권한이 있어야 하는 메서드도 있습니다. 즉 상태 저장 메서드는 이미 실행된 이전 코드 줄에 의해 메모리에 저장된 값을 사용하거나 값을 업데이트 또는 메모리에 새 값을 저장하여 애플리케이션의 상태를 수정하는 방식으로 빌드됩니다. 이 메서드를 **인스턴스 메서드**라고도 합니다.

> 상태저장(인스턴스) 메서드는 클래스에 정의된 변수인 ‘필드’ 에서 해당 상태를 추적합니다. 클래스의 새 인스턴스마다 상태를 저장할 고유한 필드 복사본이 있습니다.

> 단일 클래스에서 상태 저장 메서드와 상태 비저장 메서드를 모두 지원할 수 있습니다. 그러나 상태 저장 메서드를 호출해야 하는 경우, 메서드가 상태에 액세스 할 수 있도록 먼저 클래스 ‘인스턴스’를 만들어야 합니다.

## **1. 타입스크립트 문법**

### **변수 선언**

> 타입스크립트에서 변수를 선언할 때는 **`let`**, **`const`** 키워드를 사용합니다. 변수의 타입은 콜론(:)을 사용하여 명시할 수 있습니다.

```tsx
let name: string = 'John'; // 문자열 타입 변수
const age: number = 30; // 숫자 타입 상수
```

> 타입스크립트의 타입 시스템은 변수와 함수 등의 요소에 타입을 지정하는 것으로 이루어져 있습니다. 이를 통해 코드의 안정성과 가독성을 높일 수 있습니다.

기본적으로 타입스크립트에서 제공하는 기본 타입은 다음과 같습니다.

boolean: true 또는 false 값을 갖는 불리언 타입
number: 정수 및 실수를 모두 포함하는 숫자 타입
string: 문자열 타입
any: 모든 타입을 포함하는 타입으로, 타입을 지정하지 않은 변수는 기본적으로 any 타입입니다.
void: 값을 반환하지 않는 함수의 반환 타입
null: 값이 없음을 나타내는 타입
undefined: 값이 정의되지 않음을 나타내는 타입
never: 절대로 발생하지 않는 값의 타입
또한 배열(array), 객체(object), 함수(function) 등 다양한 타입을 지정할 수 있습니다.

변수나 함수에 타입을 명시하려면, 변수나 함수 이름 뒤에 콜론(:)과 함께 타입을 지정하면 됩니다. 예를 들어, 다음과 같이 변수에 타입을 지정할 수 있습니다.

>

```tsx
let age: number = 30;
let name: string = 'John';
let isMale: boolean = true;
```

> 함수에 타입을 명시하려면, 함수의 매개변수와 반환 값에 각각 타입을 지정하면 됩니다. 예를 들어, 다음과 같이 함수에 타입을 지정할 수 있습니다.

```tsx
function add(num1: number, num2: number): number {
  return num1 + num2;
}
```

> 타입스크립트에서는 인터페이스(interface)와 클래스(class)를 이용하여 타입을 추상화할 수 있습니다. 이를 통해 코드를 더욱 간결하고 유지보수하기 쉽게 만들 수 있습니다. 인터페이스와 클래스는 다음과 같은 방법으로 사용할 수 있습니다.

```tsx
interface Person {
  name: string;
  age: number;
}

class Student implements Person {
  name: string;
  age: number;
  grade: string;

  constructor(name: string, age: number, grade: string) {
    this.name = name;
    this.age = age;
    this.grade = grade;
  }
}
```

> 위 코드에서는 Person 인터페이스를 정의하고, Student 클래스에서 Person 인터페이스를 구현하여 name과 age 속성을 갖도록 만들었습니다. 이렇게 하면 Person 타입을 갖는 변수나 함수를 사용할 때, 이 타입에 해당하는 객체를 생성하여 전달할 수 있습니다.

### **함수 선언**

타입스크립트에서 함수를 선언할 때는 함수의 매개변수와 반환값의 타입을 명시해주어야 합니다.

```tsx
function greet(name: string): string {
  return `Hello, ${name}!`;
}
```

### **배열**

```tsx
let numbers: number[] = [1, 2, 3];
let names: Array<string> = ['John', 'Jane'];
```

### **객체**

객체를 선언할 때는 객체의 속성 이름과 속성 값의 타입을 명시해주어야 합니다.

```tsx
let person: { name: string; age: number } = {
  name: 'John',
  age: 30
};
```

### **타입 별칭**

타입 별칭을 사용하면 반복되는 복잡한 타입을 간단하게 정의할 수 있습니다.

배열을 선언할 때는 Array 또는 []를 사용할 수 있습니다. 배열의 요소 타입은 콜론(:)을 사용하여 명시할 수 있습니다

```tsx
type Person = {
  name: string;
  age: number;
};

let john: Person = {
  name: 'John',
  age: 30
};
```

## **2. 타입 시스템**

타입 시스템은 코드에서 사용되는 값의 종류를 정의하고, 해당 값이 사용될 수 있는 위치와 방법을 제한합니다. 타입 시스템을 사용하면 코드의 안정성과 가독성을 높일 수 있습니다.

### **기본 타입**

타입스크립트에서 기본적으로 제공하는 타입으로는 **`number`**, **`string`**, **`boolean`**, **`null`**, **`undefined`** 등이 있습니다.

```tsx
let num: number = 10;
let str: string = 'Hello';
let bool: boolean = true;
let n: null = null;
let u: undefined = undefined;
```

### **객체 타입**

객체 타입은 객체의 속성과 속성 값의 타입을 정의합니다.

```tsx
type Person = {
  name: string;
  age: number;
};

let john: Person = {
  name: 'John',
  age: 30
};
```

### **배열 타입**

배열 타입은 배열의 요소 타입을 정의합니다.

타입스크립트의 타입 시스템은 변수와 함수 등의 요소에 타입을 지정하는 것으로 이루어져 있습니다. 이를 통해 코드의 안정성과 가독성을 높일 수 있습니다.

기본적으로 타입스크립트에서 제공하는 기본 타입은 다음과 같습니다.

- boolean: true 또는 false 값을 갖는 불리언 타입
- number: 정수 및 실수를 모두 포함하는 숫자 타입
- string: 문자열 타입
- any: 모든 타입을 포함하는 타입으로, 타입을 지정하지 않은 변수는 기본적으로 any 타입입니다.
- void: 값을 반환하지 않는 함수의 반환 타입
- null: 값이 없음을 나타내는 타입
- undefined: 값이 정의되지 않음을 나타내는 타입
- never: 절대로 발생하지 않는 값의 타입

또한 배열(array), 객체(object), 함수(function) 등 다양한 타입을 지정할 수 있습니다.

변수나 함수에 타입을 명시하려면, 변수나 함수 이름 뒤에 콜론(:)과 함께 타입을 지정하면 됩니다. 예를 들어, 다음과 같이 변수에 타입을 지정할 수 있습니다.

```tsx
let age: number = 30;
let name: string = 'John';
let isMale: boolean = true;
```

함수에 타입을 명시하려면, 함수의 매개변수와 반환 값에 각각 타입을 지정하면 됩니다. 예를 들어, 다음과 같이 함수에 타입을 지정할 수 있습니다.

```tsx
function add(num1: number, num2: number): number {
  return num1 + num2;
}
```

타입스크립트에서는 인터페이스(interface)와 클래스(class)를 이용하여 타입을 추상화할 수 있습니다. 이를 통해 코드를 더욱 간결하고 유지보수하기 쉽게 만들 수 있습니다. 인터페이스와 클래스는 다음과 같은 방법으로 사용할 수 있습니다.

```tsx
interface Person {
  name: string;
  age: number;
}

class Student implements Person {
  name: string;
  age: number;
  grade: string;

  constructor(name: string, age: number, grade: string) {
    this.name = name;
    this.age = age;
    this.grade = grade;
  }
}
```

위 코드에서는 Person 인터페이스를 정의하고, Student 클래스에서 Person 인터페이스를 구현하여 name과 age 속성을 갖도록 만들었습니다. 이렇게 하면 Person 타입을 갖는 변수나 함수를 사용할 때, 이 타입에 해당하는 객체를 생성하여 전달할 수 있습니다.

타입스크립트에서 클래스(class)와 인터페이스(interface)는 객체 지향 프로그래밍의 기본 개념 중 하나입니다. 이를 이용하여 코드를 더욱 유지보수하기 쉽고 간결하게 작성할 수 있습니다.

### **클래스**

클래스는 객체를 생성하는데 사용되며, 이 객체는 클래스의 인스턴스(instance)라고 부릅니다. 클래스는 데이터와 해당 데이터를 조작하는 함수를 포함하며, 이를 통해 객체를 생성하고 다양한 작업을 수행할 수 있습니다.

타입스크립트에서 클래스를 정의할 때는 **`class`** 키워드를 사용합니다. 다음은 간단한 클래스의 예시입니다.

```tsx
class Person {
  name: string;
  age: number;

  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }

  greet() {
    console.log(`Hello, my name is ${this.name}. I'm ${this.age} years old.`);
  }
}

let john = new Person('John', 30);
john.greet();
```

위 코드에서는 Person 클래스를 정의하고, constructor 함수를 이용하여 name과 age 속성을 초기화합니다. 또한 greet 함수를 정의하여 객체를 인사하는 메시지를 출력합니다. 마지막으로, new 키워드를 이용하여 john이라는 Person 객체를 생성하고, greet 함수를 호출하여 인사 메시지를 출력합니다.

타입스크립트(TypeScript)는 자바스크립트(JavaScript)의 상위 집합 언어이며, 정적 타입을 지원하므로 컴파일 시 타입 검사를 수행하여 런타임 에러를 방지할 수 있습니다. 타입스크립트에서는 인터페이스(Interface)를 비롯한 다양한 개념을 제공하여 코드의 가독성과 유지보수성을 높일 수 있습니다.

인터페이스는 객체의 구조를 정의하기 위한 방법 중 하나입니다. 다음은 인터페이스의 예시입니다.

```tsx
interface User {
  id: number;
  name: string;
  email: string;
  age?: number; // 선택적 프로퍼티
  readonly created_at: Date; // 읽기 전용 프로퍼티
  [key: string]: any; // 인덱싱 가능 타입
}
```

위 코드에서 **`User`** 인터페이스는 **`id`**, **`name`**, **`email`**이라는 필수 프로퍼티와 **`age`**와 **`created_at`**이라는 선택적 프로퍼티를 가집니다. 또한, 인덱싱 가능 타입을 사용하여 다른 프로퍼티도 추가할 수 있습니다.

인터페이스는 클래스(Class)나 함수(Function)에서도 사용될 수 있습니다. 클래스에서 인터페이스를 구현하면 해당 클래스는 인터페이스에서 정의한 프로퍼티와 메서드를 모두 구현해야 합니다. 함수에서 인터페이스를 사용하면 함수의 매개변수와 반환값의 타입을 정의할 수 있습니다.

```tsx
interface Greeting {
  sayHello(name: string): void;
}

class EnglishGreeting implements Greeting {
  sayHello(name: string): void {
    console.log(`Hello, ${name}!`);
  }
}

function greet(greeting: Greeting, name: string): void {
  greeting.sayHello(name);
}

const englishGreeting = new EnglishGreeting();
greet(englishGreeting, 'John'); // 출력 결과: "Hello, John!"
```

위 코드에서 **`Greeting`** 인터페이스는 **`sayHello`** 메서드를 정의하고 있습니다. **`EnglishGreeting`** 클래스는 **`Greeting`** 인터페이스를 구현하여 **`sayHello`** 메서드를 구현합니다. **`greet`** 함수는 **`Greeting`** 인터페이스를 매개변수로 받으며, **`name`** 매개변수와 함께 **`sayHello`** 메서드를 호출합니다.
